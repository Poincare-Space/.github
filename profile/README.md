# Poincare Space

> Building the future of AI memory systems.

## About Us

We are a team of engineers, researchers, and builders working on **POINCARE** — a next-generation memory architecture for AI agents.

## The Problem

Current AI agents consume memory inefficiently. Context windows are limited, retrieval is expensive, and agents lose critical information between sessions. The memory layer is the bottleneck — not intelligence.

## Our Vision: POINCARE

POINCARE is an alternative memory system designed for autonomous AI agents. We're building:

- **Persistent Memory** — Agents remember across sessions, not just within them
- **Efficient Retrieval** — Semantic search that scales, not brute-force context stuffing
- **Agent-Native Architecture** — Memory designed for agents, not retrofitted from databases
- **Privacy-First** — Your data stays yours. Self-hosted, auditable, controllable.

## How We Work

We are a small, high-agency team that uses **cutting-edge AI agents** to accelerate development:

### Our Workflow

- **Spec-Driven Development (SDD)** — Every feature starts with a clear spec before code is written. No exceptions.
- **Agent-Assisted Development** — We leverage state-of-the-art AI agents for coding, review, testing, and documentation. Agents are tools, not replacements for engineering judgment.
- **Continuous Integration** — Automated pipelines catch issues early. Manual review catches what automation misses.
- **Async-First Communication** — We document decisions, not just discuss them. If it's not written down, it didn't happen.

### Team Principles

1. **Autonomy with Accountability** — Each engineer owns their domain end-to-end
2. **Speed with Quality** — Ship fast, but never ship broken
3. **Transparency** — All decisions, tradeoffs, and context are visible to the team
4. **Learning Culture** — We share what we learn. Every bug is a lesson, every success is a pattern
